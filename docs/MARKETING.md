# LocalForge Positioning

LocalForge turns downloaded local models into auditable software-building operators.

## Core message

Build software with your own local AI model, your own tools, and your own machine. No cloud model dependency is required.

## Differentiators

- Local-first by default.
- Works with models already downloaded into Ollama.
- Supports llama.cpp for direct `.gguf` workflows.
- Gives the model real tools rather than fake simulated actions.
- Produces auditable run logs and transcripts.
- Supports MCP tools for controlled expansion.
- Designed for technical operators, agencies, and teams that need traceability.

## Target users

- Solo builders running local coding models.
- Agencies that need reproducible AI-assisted project generation.
- DevOps teams building internal automation.
- Security-sensitive teams avoiding cloud model calls.
- AI tool developers building higher-level local agent products.

## Proof points

- `make doctor` verifies backend availability.
- `make test` verifies the harness core.
- Every run produces structured audit artifacts.
- File replacement creates backups.
- No cloud credentials are required.
